TFD-UI Documentation

TFD-UI is a declarative way to specify dynamic behaviors in html pages. Using special attributes a system of dependencies and callbacks is created which determines how the page responds to user interaction.

  1.  

Terminology

For the purposes of this document we will establish some jargon to help with things:

dependant
An html element that depends on another html element.

referant
The html element on which the dependent element depends.

Dependency Attributes

These attributes establish the dependant/referant relationship between html elements.

depends-on="name"
This element will react when the value of the element with attribute name="name" changes.
Code Demo 
<input name="ex1button" type="button" value="Click Me"/>

<p modal="alert" depends-on="ex1button">Click!</p>

Click!

Action Attributes

These attributes determine what sort of action is performed on the dependant when the referant's value changes.

depends-do="action,action,..."
Comma separated list of actions to be performed when the referant's value is changed. See the list of supported actions below:

Actions

These actions are specified as the value of the depends-do attribute of the dependant, and determine which action is to be performed.

toggle
Toggles visibility of the dependant. This is the default action, performed when no depends-do attribute is specified.

toggle-effect
Toggles visibility of the dependant using a neat little animation of some kind. Supported effects are slide and fade.

set-text
Sets the text of the dependant to the value of the referant.

set-val
Sets the value of the dependant to the value of the referant. Note that this is not always the same thing as setting the value attribute of the dependant, so this attribute should be used instead of the set-attr-value attribute.

set-attr-attr
Sets the value of the attribute attr on the dependant to the value of the referant.

scroll-top
Scrolls to the top of the page.
Code Demo 
Power:
<input name="ex2radio" type="radio" value="off" checked/>
<label>off</label>
<input name="ex2radio" type="radio" value="on"/>
<label>on</label>

<p>The power is …
  <span depends-on="ex2radio" depends-do="set-text,set-attr-class"/>
</p>

Conditional Attributes

These attributes determine when the dependant element will react to changes of the value of the referant. This is used to show/hide parts of the page in response to user interaction and things like that.

If no depends-val-* attribute is given, then the dependant element will always react to any change of the referant.

depends-val-eq="value"
This element will react when the value of the referant changes and its value is equal to value.

depends-val-neq="value"
This element will react when the value of the referant changes and its value is not equal to value.

depends-val-lt="value"
This element will react when the value of the referant changes and its value is less than value.

depends-val-le="value"
This element will react when the value of the referant changes and its value is less than or equal to value.

depends-val-gt="value"
This element will react when the value of the referant changes and its value is greater than value.

depends-val-ge="value"
This element will react when the value of the referant changes and its value is greater than or equal to value.

depends-val-re="regex"
This element will react when the value of the referant changes and its value matches the regular expression regex.

depends-val-nre="regex"
This element will react when the value of the referant changes and its value does not match the regular expression regex.
Code Demo 
Power:
<input name="ex3radio" type="radio" value="0" checked/>
<label>off</label>
<input name="ex3radio" type="radio" value="1"/>
<label>on</label>

<p>The power is …
  <span depends-on="ex3radio" depends-val-eq="0">off</span>
  <span depends-on="ex3radio" depends-val-eq="1">on</span>
</p>

Non-Form Element Values

Normally, only form elements have a value. This value can be obtained using jQuery's .val() method. TFD-UI makes it possible to interact with any element in the same way as a form element, and that element will then have a value.

To make a generic html element act like a given type of form element the acts-like="nodeName" attribute is specified on the element.

acts-like="input" type="checkbox"
This acts like a checkbox: its value toggles between 0 and 1 when the user clicks on it.

acts-like="input" type="radio"
This acts like a radio button: only one of these elements (identified by name attribute) can be active at any time. The checked element is set when the user clicks on the element. The value of the set of radio buttons is the value of the checked element.

acts-like="input" type="button"
This acts like a button: its value is constant, but its onchange handler is triggered whenever the user clicks on the element.

acts-like="input" type="hidden"
This acts like a hidden form element: its value is constant and it does not respond to user interaction, at all.

acts-like="select"
This acts like a select element. It allows the user to select an option from a list of mutually exclusive choices. When the user clicks on the element the choices are displayed. If the user clicks on a specific option, the select element takes the value of that option. This element should contain a number of acts-like="option" elements representing the various choices the user can select. The selected element will be shown, and the rest hidden.

acts-like="option"
This acts like an option element, representing one of the choices of a select element. The selected attribute, if present, indicates that a particular option is currently selected.

options-for="select"
This specifies that this element should be displayed as the dropdown when the user clicks on the select element identified by name="select". This element should contain a number of acts-like="option" elements representing the various choices the user can select.
Code Demo 
<label>Power is:</label>
<span acts-like="select" name="ex4select">
  <span acts-like="option" value="0" style="color:green">OFF</span>
  <span acts-like="option" value="1" style="color:red">ON</span>
</span>
<span options-for="ex4select">
  <span acts-like="option" value="0" selected>Turn off</span>
  | <span acts-like="option" value="1">Turn on</span>
</span>

<p modal="alert" depends-on="ex4select" depends-val-eq="1">
  Power is now on!
</p>

Power is now on!

Forms

Normally, forms are used to collect information from the user which is then submitted via GET or POST request to the server. TFD-UI disables the standard form behavior and wires up the form elements to use AJAX, instead.

Form elements in TFD-UI pages are the linkage between the data and the user interface. Forms can be configured to make AJAX requests that can automatically fill in inline templates on the page (see the TEMPLATES section, below).

Form Action Attributes

These attributes determine how the action attribute of the form is to be interpreted and how the result is to be processed. The action will be initiated any time an input element contained in the form changes its value. The result of the action must be JavaScript data structures, and the form's value is set to this JavaScript data when the action is successfully completed.

form-do="action,action,..."
Comma separate list of actions to be performed when any of the form's input element's values are changed. See the list of supported actions below:

Form Actions

These actions are specified as the value of the form-do attribute of the form, and determine which type of action is performed when one of the form's input elements changes.

get-json
Makes a GET request to the url specified in the form's action attribute, and processes the response as JSON data. The value of the form element is set to the result of parsing JSON response. This is the default action, performed when no form-do attribute is specified.

parse-page
This action parses the DOM and processes it using some submodule (specified with the action attribute), which sets the form's value. See the list of supported submodules below:

Page Parsing Submodules

These submodules are used in conjunction with the parse-page form action. They process the page's DOM and set the form's value.

outline
This submodule produces an array of JavaScript objects representing the document outline (i.e. a list of headings and url fragments suitable for use with anchor tags). The table of contents of this page uses this submodule.

    </select>
  </form>

Form Template Filling

Forms can be used to fetch data from the server and use that data to fill inline templates in the page. (See the TEMPLATES section below.) The templates are filled when the value of the form itself changes (usually as a result of the action being performed).

fill-template="template,template,..."
Fills in the inline templates in the comma separated list of template names. The inline templates are identified by the attribute template="template".

Templates

TFD-UI supports inline templates. Inline templates are html elements that are in the page, but contain dummy content. The content is intended to be filled in later with data obtained from some source, usually as a result of an AJAX request to the server.

Templates are filled in based on the data-fill and data-fill-* attributes on the template element and its children. The data must be an array of JavaScript objects (POJOs). This array is iterated through, and for each object a clone of the template is made and filled in as specified using the attributes below:

template="template"
Identifies this element as an inline template, filled with dummy content. The value of this attribute is what is references when the template is filled, as a result of a form action, for example.

data-fill="prop"
Sets the text of this element to the value of the prop property of the JavaScript object.

data-fill-attr="prop"
Sets the value of this element's attr attribute to the value of the prop property of the JavaScript object.
Code Demo 
<form action="data.json" fill-template="ex6-template">
</form>

<table cellpadding="6px" cellspacing="0px">
  <thead>
    <tr>
      <th>Item</th>
      <th>Price</th>
    </tr>
  </thead>
  <tbody>
    <tr template="ex6-template">
      <td data-fill="animal">Fish</td>
      <td data-fill="price" align="right">1.00</td>
    </tr>
    <tr template="ex6-template">
      <td data-fill="animal">Mammal</td>
      <td data-fill="price" align="right">5.00</td>
    </tr>
  </tbody>
</table>

Modal Dialogs

These attributes are used to transform any html element into a modal dialog that will be hidden when the page loads, and displayed in response to some interaction with a referant.

modal="type"
This element will be converted into a modal dialog of type type. The title of the dialog is specified by adding a title attribute to the element. See the list of modal dialog types below:

Modal Dialog Types

The different modal dialog types contain various configurations of buttons and things.

plain
This type of dialog has no buttons, at all.

confirm
This type of dialog has two buttons: "yes" and "no". Additionally, the element has a value, which is set to 0 if the user clicks "no", or 1 if they click "yes".

alert
This type of dialog has one button: "ok".
Code Demo 
<input type="button" name="ex7button1" value="Plain"/>
<input type="button" name="ex7button2" value="Confirm"/>
<input type="button" name="ex7button3" value="Alert"/>

<p modal="plain" title="Plain Dialog" depends-on="ex7button1">
  Hello, world!
</p>
<p modal="confirm" title="Confirmation Dialog" depends-on="ex7button2">
  Hello, world!
</p>
<p modal="alert" title="Alert Dialog" depends-on="ex7button3">
  Hello, world!
</p>
